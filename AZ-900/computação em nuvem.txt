MODULO 2:

Região é uma localização física onde vc pode ter 1 ou 2 ou mais datacenters e uma região.
- azure é o provedor que mais oferece regiões globais do que qualquer outro provedor
- mais de 60 regiões e representa mais de 40 países
- as regiões são composta por 1 ou mais 2 datacenter muito proximos - para que se caso um entre em inatividade, tem o outro
- você consegue ter redundancias de dados nas suas aplicações

pares de região:
são pares que possuem 2 regiões com uma certa distância
- minimos 300 milhas de separação entre pares de regiões
- replicação automatica para alguns serviços
- recuperação e região priorizada em caso de interrupção
- As atualizações são lançadas sequencialmente para minimizar o tempo de inatividade
- caso, tenha uma inatividade de uma região no brasil, o seu serviço vai rodar no estados unidos, então é bom, você conhecer a região par da sua.

Opções de disponibilidade

SLA VM
- 99,9% com armazenamento Premium, cerca de 40 minutos por mês de inatividade
- VM unica
- lift-and-shift mais fácil

SLA VM
99,99%
zona de disponibilidade
proteção contra falhas de datacenter inteiro
caso, você queira aumentar a disponibilidade dessa VM, você pode usar uma zona de disponibilidade

zona de disponibilidade
- oferece proteção contra tempo de inatividade, so um datacenter falhe
- separa fisicamente os datacenters dentro da região
- os datacenter é equipado com alimentação necessario, resfriamento de rede independentes
- conectado pormeio de redes privadas de fibra óptica

Serviços de rede

- rede virtual (Vnet) do azure é uma rede virtual que permite que você se conecte a internet e rede locais
- gateway de rede virtual privada(VPN) é um tubo criptográfico entre a rede da azure e um local na internet publica
- express route do azure amplia redes locais para o azure por meio de uma conexão privada facilitada por um provedor de conectividade


serviços de armazenamento:
- armazenamento de container (blob) é otimizado para armazenar grandes quantidades de dados não estruturado com dados binários ou de texto, você pode guardar imagens, PDFs etc

-armazenamento de disco igual um ssd e hd físico só que na nuvem

- Arquivos do azure é meio que uma pasta virtual, você configura compartilhamentos de arquivos de rede altamente disponíveis que podem ser acessados usando o protovolo (SMB)

CAMADAS DE ACESSO DE ARMAZENAMENTO DO AZURE

Frequente: arquivos acessados com grande frequência
Esporádico: otimizada para arquivos com pouca frequência de acesso, armazenado pelo menos há 30 dias, para ler é mais caro
Arquivar: otimizada para armazenamento de dados acessados raramente ,pelo menos 180 dias com requisitos de latência flexíveis, quase não conta armazenamento e mais cobra caro para ler

Seviço de banco de dados da azure

Azure cosmo database é um serviço de banco de dados distribuído globalmente que escala de maneira elástica e independente a taxa de transferência e o armazenamento

Instancias gerenciada do SQL do azure permite clientes existentes do SQL server façam "lif and shift" dos aplicativos locais para a nuvem com mínimo de alterações no aplicativo e no banco de dados
- plataforma como serviço sempre atualizada
- preserva todos os recursos de PAAS(atualizações de versão e aplicação de patch automáticos, backups autoatizados e alta disponibilidade)
-troca as licenças existentes para taxas descontadas na instancia gerenciada de SQL usando o benefício híbrido do azure
